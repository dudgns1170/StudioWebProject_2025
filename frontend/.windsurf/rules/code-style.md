## General Code Style & Formatting

- 모든 코드는 영어로, 문서는 한국어로 작성한다.
- 모든 변수와 함수(매개변수, 반환 타입 포함)에 타입을 명시한다.
- any 사용을 피한다.
- 필요한 타입은 반드시 생성하여 사용한다.
- 공개된 클래스와 메서드는 JSDoc으로 문서화한다.
- 함수 내부에는 불필요한 빈 줄을 두지 않는다.
- 파일 하나당 하나의 export만 사용한다.

## Functions & Logic

- 함수는 한 가지 목적만 수행하며 20줄 이하로 유지한다.
- 깊은 중첩을 피하기 위해 다음을 사용한다:
  - early return
  - 유틸리티 함수로 로직 분리
- map, filter, reduce 같은 고차 함수를 적극적으로 사용해 로직을 단순화한다.
- 간단한(<3줄) 로직은 화살표 함수를, 그 외에는 명명된 함수를 사용한다.
- null/undefined 체크 대신 기본 매개변수를 활용한다.
- 여러 매개변수 또는 데이터 반환 시 RO-RO 패턴(Receive Object, Return Object)을 사용한다.

## Data Handling

- 원시 타입 남용을 피하고, 필요한 경우 복합 타입으로 감싼다.
- 함수 내부에 검증 로직을 넣지 말고, 검증이 필요한 경우 별도 클래스 내부에서 처리한다.
- 데이터는 가능한 불변성을 유지한다:
  - 변경 불가능한 속성은 readonly 사용
  - 변경되지 않는 리터럴은 as const 사용
